# v1.0.0 — Stable explicit-probe concurrency testing (preview)

[Index](roadmap.md) · [v0.9.0](v0.9.0-roadmap.md)

**Status:** preview.

## Theme

Stable 1.0 for deterministic concurrency testing with explicit probes, replay tokens, and bounded exploration (if shipped in 0.6+).

## Main outcome

- Semver-stable `RunAsync` / schedule / replay token story
- Documented non-goals remain unchanged
- Three featured examples + workflow docs as the default onboarding path

## Must not do

- Automatic await interception as a default feature
- Coyote replacement positioning
- Rider plugin or UI as a 1.0 requirement
- IL rewriting or `TaskScheduler` replacement

## Success criteria

- New users can reproduce a race with a replay token in ordinary xUnit tests
- Random-only limits remain honest in docs and APIs
- Exploration (if present) is bounded and opt-in
